Design principles

Built for repeated work, not a marketing dashboard.

Local first

The app listens on 127.0.0.1. Your library and clipboard history stay on your Mac unless you explicitly export or share them.

Summary before full content

Agents discover lightweight summaries first. Full skills, prompts, MCP configs, and clipboard content are loaded only by id when needed.

Quiet native UI

A compact macOS panel, readable spacing, keyboard-first clipboard actions, and no noisy default libraries.

Built-in MCP bridge

Install one MCP server, manage the rest in DingDong.

The built-in bridge exposes a small set of tools for summary-first resource discovery, safe skill loading, MCP recommendation, native MCP config installation, and finish notifications.

Agent config

Add this MCP server to Codex or Claude Code after installing DingDong.

{
  "mcpServers": {
    "dingdong": {
      "command": "/Applications/DingDong.app/Contents/MacOS/dingdong-mcp"
    }
  }
}
dingdong_bridge

Start here. Returns task-scoped summaries, recommended resources, and privacy rules.

dingdong_search_assets

Search prompts, skills, MCP references, notes, and other resources by query and type.

dingdong_get_asset

Fetch one resource by id. Summary mode hides heavy content; full mode is explicit.

dingdong_load_skill

Load a selected skill resource only after an agent decides it is relevant.

dingdong_recommend_mcp

Find MCP server references that match the current task.

dingdong_install_native_mcp

Dry-run or install a stored MCP reference into Codex or Claude Code config.

dingdong_notify

Trigger the DingDong sound, unread badge, and menu bar notification icon.

Loopback HTTP API

Readable endpoints for agents and scripts.

DingDong also exposes local HTTP routes for system status, resource search, clipboard insights, task sessions, durable memories, handoff notes, and UI deep links.
